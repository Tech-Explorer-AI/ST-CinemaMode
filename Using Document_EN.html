2. Options Format

Options must be wrapped with [[[ and ]]]:

[[[
Choose your action:
A. Option content one
B. Option content two
C. Option content three
D. Option content four
]]]

Supported Option Formats:

Format Example
[A] Text [A] Play games with Zhang Lei
A. Text A. Play games with Zhang Lei
**A.** Text **A.** Play games with Zhang Lei
A) Text A) Play games with Zhang Lei
A、Text A、Play games with Zhang Lei

💡 Usage Details

1. Character Portrait Priority

The system loads character portraits in this priority order:

  1. Mood folder portraits - images/portrait/library/[Gender]/[Name]/[Mood].png
  2. Exact match portraits - images/portrait/[Name].png
  3. Gender folder portraits - images/portrait/library/[Gender]/[number].png
  4. Universal portraits - images/portrait/library/Universal/[number].png
  5. Default placeholder - 👤 emoji

❓ Frequently Asked Questions

Q: Character portraits don't display?
A: Check:
  • Images are in the correct folder (images/portrait/library/Female/ or images/portrait/library/Male/)
  • File names match exactly (case-sensitive for English, Chinese names work)
  • Image format is supported (PNG recommended)
  • Character name appears in the Relationships section of the status bar
Q: Music doesn't play?
A: Check:
  • Music file is MP3 format
  • File is in images/music/ directory
  • Browser allows autoplay
  • Music file name matches the mapping in MUSIC_MAP
Q: Status bar isn't parsed?
A: Check:
  • Wrapped with ``` (three backticks)
  • Format is correct (uses colon ： or :)
  • Section names are in the keyword list
  • Has ```markdown before the status bar
Q: Options don't appear?
A: Check:
  • Wrapped with [[[ and ]]]
  • Option format is correct (A. B. C. D.)
  • Options are on separate lines
Q: Background image doesn't display?
A: Check:
  • Image is in images/background/ folder
  • File name is mapped in SCENE_FILE_MAP
  • Scene name matches (extracted from the 📍 location in the status bar)
  • Image format is supported
Q: Affection displays negative values?
A: Use the format Affection:-value/100 in the status bar to display negative affection.
